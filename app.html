Phase 1: Pre-Migration & Setup Details

📢
Comprehensive SDK Integration

Integrate the SDK into iOS & Android apps. Meticulously map key in-app events like `purchase` and `add_to_cart`.

💾
Historical Data Import

Work with our team to import existing user attribution and cohort data to preserve your user history.

💻
Dashboard & Team Training

Configure your dashboard with custom reports and conduct comprehensive training for your marketing and data teams.

Phase 2: Testing & Validation Details

🔗
Parallel Tracking

For a brief period, run both MMPs simultaneously to compare attribution data directly and ensure accuracy.

🔬
A/B Testing New Campaigns

Launch small, controlled campaigns using only AppsFlyer links to validate the accuracy of install and event attribution.

Phase 3: Full Migration & Decommissioning Details

🔒
Update All Tracking Links

Systematically update all active campaign tracking links (paid media, owned channels, partners) to new AppsFlyer links.

💁
Partner Communication

Ensure all ad network and media partners are informed of the switch and have the correct new tracking links.

🚀
Sunset Old SDK

After a successful transition, the old SDK can be safely removed from the QuickEats app in a future update.
